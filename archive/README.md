# Historical Archive

Provenance-only snapshots of earlier framework drafts. Not part of the active codebase. Retained for citation continuity and as reference for material that was triaged into or out of the canonical framework.

## Contents

| File | What | Why retained |
|---|---|---|
| `v9_1_deferred_ideas.md` | List of ideas explored in v9.x → v10.0 transition; some triaged out, some retained, some flagged with critical errors found and corrected | Documents the dimensional-error history and the explicit "what is allowed / not allowed to claim" discipline that informed the the current framework tier hierarchy |
| `v9_1_changelog.md` | Section-by-section changelog from v9.0 → v9.1 | Shows the maximalist phase before v10's triage; useful for understanding which the current framework T4 modules originate from v9.x and what corrections were applied |

| `GSC_Framework.md.legacy` | The v11.0.0 canonical theoretical framework (released as v10.1.1-late-time-r4) | The immediate predecessor; the current framework's GSC_Framework.md is its layered-architecture successor |
| `GSC_Framework.tex.legacy` | LaTeX source for the v11 framework | For citation cross-checks |

## Usage discipline

Material in this directory **may be referenced** for historical context. It **must not** be cited as the current canonical position of the framework. The current canonical position is in `../GSC_Framework.md`.

If material from the archive is to be promoted back into the active framework, it must:

1. Pass dimensional-consistency checks (see `v9_1_deferred_ideas.md` for the critical-error log);
2. Be assigned an explicit tier label (T1–T4);
3. Carry an explicit kill-test;
4. Be referenced from `../GSC_Framework.md` with its tier and status.
