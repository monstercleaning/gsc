# Legacy Documents Archive

Documents retained from the predecessor framework cycle that are no longer part of the active documentation set. They fall into three categories:

## Release-cycle artifacts

Submission checklists, JOSS preflight notes, paper-2 build documentation, and the SBOM tied to a specific release. Examples:

- `ARXIV_*.md`, `JOSS_*.md`, `PAPER2_*.md` — submission infrastructure for the immediate predecessor release.
- `SBOM.md` — software bill of materials for the v11.0.0 release.
- `submission_runbook.md`, `status_canonical_artifacts.md`, `LEGACY_VERSIONED_ARTIFACTS.md` — operator-facing release status.
- `external_reviewer_feedback.md` — reviewer dialog from the v10.1 cycle.

These will be replaced by the corresponding v12-cycle materials once Papers A–D are submitted.

## Roadmap snapshots

- `GSC_Consolidated_Roadmap_v2.5.md`, `v2.8.md`, `v2.8.1_patch.md` — older roadmap drafts. The current canonical roadmap is in [../../GSC_Framework.md §13](../../GSC_Framework.md).

## Phase-specific status (`phase_specific_status/`)

Working notes from Phase-2 (E2 CMB closure) and Phase-3 (structure formation) iterations. They capture the methodology and intermediate findings of the Phase-2/3 effort and are useful as reference for anyone restarting that work, but are not part of the canonical framework presentation.

## Discipline

Material here may be cited for historical context. It must not be treated as the current framework's position. The canonical position is in [../../GSC_Framework.md](../../GSC_Framework.md).

To restore a document to the active set:

1. Re-read it for v12 compatibility (paths, claims, tier alignment);
2. Move it back to `../../docs/` with appropriate edits;
3. Reference it from [../../GSC_Framework.md](../../GSC_Framework.md) or another active doc.
