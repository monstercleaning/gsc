# GSC — Consolidated Roadmap v2.8.1 Patch (Referee‑safe wording)

**Applies to:** `docs/GSC_Consolidated_Roadmap_v2.8.md` (verbatim copy of Roadmap v2.8 “Final”).

**Purpose:** Clarify roadmap wording around **DESI “DR2”** so it remains accurate and referee‑safe regardless of how DESI labels (a) full “Data Release” portals vs (b) cosmology/BAO “products” used in inference tooling.

**Scope:** This patch is **wording/terminology only**. It does **not** change the project plan, milestones, or technical requirements.

---

## Patch 1 — DESI DR2 wording in Part L

### Where
In Roadmap v2.8, **Part L: Data / Software Dependencies** table, row:
- **Dependency:** “DESI BAO likelihood”

### What to change (recommended replacement text)
Replace the DR2 sentence with the following neutral wording:

> **Use DR1 as baseline (matches E2 context).** Include **DR2 BAO/cosmology products** as an **additional robustness check** whenever they are **public** and **supported by the chosen likelihood tooling**. **Avoid fixing a calendar date** in the roadmap.

### Interpretation note
In Roadmap v2.8, any mention of **“DR2”** should be read as:
- **“DR2 BAO/cosmology products (for inference)”**,
and **not necessarily** as:
- “DESI’s full DR2 portal / full‑dataset release page.”

**End of patch.**
