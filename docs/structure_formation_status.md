# Structure Formation Status (Bridge v1)

## What exists in the current framework today

- Late-time background and measurement-model infrastructure are implemented and reproducible.
- Early-time Phase-2 E2 diagnostics use compressed CMB priors as a diagnostic layer.
- Full perturbation-level structure-formation modeling is not yet part of the canonical pipeline.

## Structure bridge added in this milestone

- `gsc/structure/transfer_bbks.py`: BBKS transfer approximation with Sugiyama baryon-corrected shape parameter.
- `gsc/structure/growth_factor.py`: GR-baseline linear growth solver (`D(z)`, `f(z)`) driven by an explicit `H(z)` history.
- `scripts/phase2_sf_structure_report.py`: deterministic report tool that emits `T(k)` samples and growth summaries.
- `scripts/phase2_sf_fsigma8_report.py`: deterministic linear-growth `fσ8` diagnostic with optional analytic `σ8` profiling against user CSV data.

These are approximation-first diagnostics and are not a Boltzmann hierarchy replacement.

## Scope note: linear-theory only; dark matter not solved

- This module is a linear-theory diagnostic channel for consistency checks.
- It is not a full perturbation-theory closure and not a survey-complete LSS likelihood.
- It does not claim dark-matter elimination; baseline tests keep standard matter-content assumptions.
- Canonical scope and roadmap boundaries are tracked in:
  `docs/project_status_and_roadmap.md`.

## Dark Matter stance (current release)

- The baseline keeps standard matter components in the background budget.
- `Omega_m` is interpreted as the standard matter sector used in the current diagnostic setup (baryons plus cold dark matter in the baseline picture).
- The current release does not make a claim that dark matter is removed from cosmological modeling.

## What is still missing

- Full perturbation system for modified gravity / scalar-sector dynamics.
- Full CMB peak-level and LSS likelihood coupling to the same parameterization.
- Nonlinear structure diagnostics (e.g., N-body level) and survey-level inference.

## Linear growth diagnostic (`fσ8`)

Current implementation adds a minimal linear-growth layer over explicit `H(z)` histories:

- GR linear growth equation in `ln(a)` with deterministic RK4 integration.
- Derived observables: `D(z)`, `f(z) = dlnD/dlna`, `g(z)=fD`.
- Optional `σ8` nuisance profiling (analytic, diagonal covariance) from user-provided CSV (`z,fsigma8,sigma`).

This remains a diagnostic approximation. It is not a full perturbation solver, not a Boltzmann hierarchy, and not a survey-complete LSS likelihood fit.

EdS sanity run:

```bash
python3 scripts/phase2_sf_fsigma8_report.py \
  --history lcdm --H0 70 --Omega-m 1 --Omega-lambda 0
```

Fit/profiling run with user CSV:

```bash
python3 scripts/phase2_sf_fsigma8_report.py \
  --history lcdm --H0 70 --Omega-m 0.3 --Omega-lambda 0.7 \
  --data my_fsigma8.csv --format json --json-out fsigma8.json
```

## Primordial -> sigma8 bridge (baseline)

M83 adds a deterministic baseline bridge from primordial parameters to linear
growth amplitude:

- Primordial input: `(A_s, n_s)` with default pivot `k0=0.05 1/Mpc`.
- Transfer baseline: BBKS + Sugiyama shape parameter (default), with optional
  EH98 no-wiggle transfer backend.
- Growth baseline: GR linear growth (`D(z)`, `f(z)`) driven by the selected
  background `H(z)`.
- Derived amplitude path: `A_s,n_s -> P(k) -> sigma8(0) -> sigma8(z), f*sigma8(z)`.

This bridge is approximation-first and diagnostic-only. It is not a claim of
full perturbation closure, not a Boltzmann hierarchy replacement, and not a
freeze-frame perturbation derivation.

## Primordial tilt and pivot knobs (`n_s`, `k_pivot`)

M91 makes primordial assumptions explicit in the linear bridge:

- `n_s` is an explicit input (default `1.0` for backward-compatible baseline).
- `k_pivot` is explicit in `1/Mpc` (default `0.05`).
- report JSON now carries `primordial_ns` and `primordial_k_pivot_mpc` metadata
  so downstream comparisons can audit the exact primordial setup.

Example:

```bash
python3 scripts/phase2_sf_fsigma8_report.py \
  --history lcdm --H0 70 --Omega-m 0.3 --Omega-lambda 0.7 \
  --sigma8-mode derived_As --As 2.1e-9 \
  --ns 0.965 --k-pivot 0.05 --format json
```

## Transfer function approximations (BBKS vs EH98 no-wiggle)

- `bbks` (default): compact approximation with Sugiyama baryon-corrected shape
  parameter. Useful for fast diagnostics.
- `eh98_nowiggle` (optional): Eisenstein & Hu 1998 smooth no-wiggle transfer,
  including baryon suppression through effective shape scaling.
- Both options are approximation backends for linear diagnostics and are not
  CAMB/CLASS-equivalent Boltzmann transfer solvers.
- CLI selection is explicit via:
  `--transfer-model {bbks,eh98_nowiggle}` (alias `eh98` accepted).
- Approximation references:
  - BBKS: Bardeen, Bond, Kaiser, Szalay (1986), transfer-function fitting formula.
  - EH98 no-wiggle: Eisenstein & Hu (1998), smooth baryon-suppressed transfer approximation.

Run in nuisance-amplitude mode (backward-compatible):

```bash
python3 scripts/phase2_sf_fsigma8_report.py \
  --history lcdm --H0 70 --Omega-m 0.3 --Omega-lambda 0.7 \
  --data my_fsigma8.csv --format json --json-out fsigma8_nuisance.json
```

Run in derived-amplitude mode (`sigma8` from `A_s,n_s`):

```bash
python3 scripts/phase2_sf_fsigma8_report.py \
  --history lcdm --H0 70 --Omega-m 0.3 --Omega-lambda 0.7 \
  --sigma8-mode derived_As --As 2.1e-9 --ns 0.965 \
  --transfer-model eh98_nowiggle \
  --kmin 1e-4 --kmax 1e2 --nk 2048 \
  --data my_fsigma8.csv --format json --json-out fsigma8_derived_as.json
```

CSV input format for `--data` is:

- header: `z,fsigma8,sigma`
- optional comments: lines starting with `#`
- diagonal-variance chi2 in the current tool version

## RSD fσ8 diagnostic (stdlib-only)

M84 adds an opt-in RSD diagnostic channel in
`scripts/phase2_sf_fsigma8_report.py`:

- built-in dataset:
  `data/structure/fsigma8_gold2017_plus_zhao2018.csv`
- optional external dataset via `--rsd-data`
- diagonal-covariance chi2 summary
- analytic sigma8 profiling in nuisance mode (linear amplitude profile)
- optional AP-like correction (`--rsd-ap-correction ref_omega_m_lcdm`) as an
  approximation-first toggle

This remains a diagnostic baseline and not a full structure-formation fit.
Specifically:

- GR linear growth + BBKS transfer remain approximation-first.
- AP correction is an optional approximation and not a survey-complete RSD
  pipeline.
- No claim is made that this resolves dark matter or completes perturbation
  closure.

Quick run with the built-in RSD dataset:

```bash
python3 scripts/phase2_sf_fsigma8_report.py \
  --history lcdm --H0 70 --Omega-m 0.3 --Omega-lambda 0.7 \
  --sigma8-mode nuisance \
  --rsd --format json --json-out rsd_fsigma8_diag.json
```

Derived-amplitude run (`sigma8(0)` from `A_s,n_s`) with optional AP-like
correction:

```bash
python3 scripts/phase2_sf_fsigma8_report.py \
  --history lcdm --H0 70 --Omega-m 0.3 --Omega-lambda 0.7 \
  --sigma8-mode derived_As --As 2.1e-9 --ns 0.965 \
  --rsd --rsd-ap-correction ref_omega_m_lcdm \

```

## Phase-2 integration: RSD overlay on E2 best candidates

M85 wires the same linear-growth RSD diagnostic into
`phase2_e2_best_candidates_report.py` as an additive overlay for top-N E2
candidates:

- computes per-candidate `chi2_rsd` with analytic nuisance-`sigma8` profiling
  (or optional derived-amplitude mode),
- reports combined ranking via `chi2_total_plus_rsd`,
- emits deterministic paper snippet files `phase2_sf_rsd_summary.{md,tex}` for
  Phase-2 bundle/paper wiring.

This remains a diagnostic layer only:

- linear GR growth over explicit `H(z)` backgrounds,
- standard CDM+baryon baseline assumptions,
- full freeze-frame perturbation derivation and survey-complete structure
  likelihoods are deferred.
- no claim is made that dark matter is eliminated.

Quick run:

```bash
python3 scripts/phase2_e2_best_candidates_report.py \
  --input /path/to/merged.jsonl \
  --top-n 10 \
  --sf-rsd \
  --format json
```

## Integration into E2 scan records (M89)

`phase2_e2_scan.py` can now emit additive RSD proxy fields directly in JSONL
with `--rsd-overlay`:

- `rsd_overlay_ok`, `rsd_overlay_skip_reason`
- `rsd_dataset_id`, `rsd_dataset_sha256`
- `rsd_n`, `rsd_chi2`, `rsd_sigma8_0_best`, `rsd_dof`
- `rsd_ap_correction`, `rsd_mode`

Interpretation: `rsd_chi2` is the profile-min chi2 over nuisance `sigma8_0`
for the configured RSD dataset and linear-GR growth model. This remains an
operational diagnostic, not a full perturbation/LSS likelihood closure.

## RSD overlay configuration knobs in Phase-2 scans (M95)

`phase2_e2_scan.py` keeps RSD overlay optional (`--rsd-overlay`) and now exposes
explicit model knobs for reproducibility/auditability:

- `--rsd-transfer-model {bbks,eh98_nowiggle}`
- `--rsd-ns <float>`
- `--rsd-k-pivot <float>` (alias: `--rsd-k0-mpc`)

Scope semantics:

- these knobs are **effective** only for `--rsd-mode derived_As`;
- in nuisance/profile mode (`--rsd-mode profile_sigma8_0`), they are recorded as
  non-effective and do not change the scan configuration hash;
- this keeps shard merge guardrails (`scan_config_sha256`) deterministic without
  false mismatches from irrelevant settings.
