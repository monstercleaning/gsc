# Redshift Drift Beyond FLRW (Systematics Note)

**Status:** diagnostic / referee-defense note (out of submission scope for the current framework core late-time claims).  
**Purpose:** address the common reviewer objection: "you are using the FLRW Sandage--Loeb redshift-drift formula; what about inhomogeneities?"

This note is **not** introducing new model claims. It summarizes how redshift drift is defined in a general spacetime, and how radially inhomogeneous models (e.g. LTB) can produce different drift signatures compared to FLRW background expectations.

## 1. Definition (General Spacetime)

Operationally, **redshift drift** is the rate of change of the observed redshift of a given source as measured by an observer, when both are followed along their worldlines and the light-ray is re-observed at two nearby observer times. In any metric theory, this is a well-defined observable, but its mapping to a compact background formula depends on geometry and kinematics assumptions.

In FLRW, for a comoving source and comoving observer, one obtains the familiar Sandage--Loeb relation:

\[
\dot z = H_0(1+z) - H(z).
\]

Outside FLRW, the drift generally depends on the detailed spacetime structure along the line of sight (expansion, shear, acceleration, etc.), and the FLRW relation is not guaranteed to hold without additional assumptions.

## 2. Inhomogeneous (LTB) Example: Different Drift Behavior Is Possible

In radially inhomogeneous Lema\^{\i}tre--Tolman--Bondi (LTB) models, redshift drift can differ significantly from FLRW expectations and can vary with observer position and the inhomogeneity profile. This is well known in the literature:

- Yoo et al. discuss redshift drift signatures in LTB universes and how they differ from FLRW intuition.  
  Reference: Yoo et al., "Redshift drift in LTB universes" (see linked note / preprint copy).  
- Codur & Marinoni (PRD 2021) provide a detailed analysis of redshift drift in radially inhomogeneous cosmologies and emphasize qualitative differences relative to \(\Lambda\)CDM/FLRW.  

The key point for reviewer purposes: **a measured drift sign/amplitude is still an observable**, but the clean mapping "drift sign directly discriminates FLRW histories" is only as robust as the background/kinematic assumptions used in that mapping.

## 3. How This Interacts With the current framework Scope

The the current framework late-time submission remains **background-level and measurement-model driven** (post-recombination). Under the current Roadmap v2.8 framing, drift is used as a supporting no-go diagnostic / historical motivation, not as the primary project discriminator.

This note should be read as a **systematics caveat**:

- If the real Universe has significant line-of-sight inhomogeneity effects for the relevant drift measurements, then a full interpretation would require extending the measurement model beyond a single background \(H(z)\) mapping.
- That extension is **out of scope** for the the current framework late-time submission and would be treated as a separate diagnostic/systematics module.
